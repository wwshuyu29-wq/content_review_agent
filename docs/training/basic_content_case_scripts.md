# 基础内容校对训练脚本

更新时间：2026-07-16

用途：给内部团队和后续模型训练提供最小可用样例。当前 AI 只辅助发现基础错误、品牌名错误、低级语病、重复表达、绝对化表述和产品能力越界；实测可信度由人工审核，不进入自动评分维度判断。

## 使用方式

1. 先用 `data/review_cases/team_audit_cases.json` 做端到端测试，确认看板能沉淀工作量、通过率和问题聚类。
2. 用 `data/training/basic_content_error_script.jsonl` 测试基础错误识别能力。
3. 用 `data/training/basic_content_correct_script.jsonl` 测试模型是否会误报正确内容。
4. 每次新增样例时必须填写 `project_code`、`text`、`labels` 和 `dimension_focus`，方便后续按项目和问题类型统计。

## 错误案例覆盖

- 品牌名错误：小度想像 -> 小度想想。
- 错别字：规化 -> 规划。
- 重复表达：同一句内重复“规划路线”。
- 标点错误：重复逗号。
- 绝对化承诺：100%、唯一最优解。
- 未确认产品能力：自动订酒店、判断哪家最划算。

## 正确案例标准

- 品牌名写作“小度想想”。
- 功能描述只来自项目 Brief 或已确认口径。
- 主观体验要有边界，比如“我使用后感觉”“出门前再结合当天路况确认”。
- 不使用“100%”“唯一最优”“所有用户都适用”等保证式表达。
- AI 生成内容必须保留人工复核提示，未确认能力不写入正文。
