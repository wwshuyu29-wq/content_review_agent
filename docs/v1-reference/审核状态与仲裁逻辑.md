# 审核状态与结果仲裁逻辑

六个 Agent 只负责输出问题和建议，不直接修改内容最终状态。最终状态由后端仲裁器根据固定优先级计算。

---

## 一、单 Agent 可用结论

```text
PASS
PASS_WITH_SUGGESTIONS
AUTO_FIX_PENDING_CONFIRMATION
NEED_TEXT_FIX
NEED_MEDIA_FIX
HUMAN_REVIEW
BLOCK
```

---

## 二、最终状态优先级

从高到低：

```text
BLOCK
> HUMAN_REVIEW
> NEED_MEDIA_FIX
> NEED_TEXT_FIX
> AUTO_FIX_PENDING_CONFIRMATION
> PASS_WITH_SUGGESTIONS
> PASS
```

只要任一 Agent 输出更高优先级状态，最终状态不得被低优先级 Agent 覆盖。

---

## 三、仲裁规则

### 1. BLOCK

触发：

- 明确虚构产品功能；
- 明确伪造测试证据；
- 重大误导；
- 明确泄露敏感信息且无法修复；
- 项目明确禁止的内容。

状态：

```text
review_status = REJECTED
publish_status = NOT_READY
```

### 2. HUMAN_REVIEW

触发：

- 未确认功能；
- 证据不足；
- 资料冲突；
- 竞品比较材料不足；
- 隐私、授权、人物或舆情问题；
- 模型置信度低于阈值；
- 同一问题被不同 Agent 给出冲突结论。

状态：

```text
review_status = HUMAN_REVIEW_REQUIRED
```

### 3. NEED_MEDIA_FIX

触发：

- 图片或视频泄露隐私；
- 素材来源不明；
- OCR文字与正文不一致；
- 视频中出现需修改字幕或画面；
- 文件规格不合格。

状态：

```text
review_status = SUPPLIER_REVISION_REQUIRED
open_tasks += MEDIA_FIX
```

### 4. NEED_TEXT_FIX

触发：

- 无依据绝对化；
- 品牌名称或定位错误；
- 标题与正文不一致；
- 测试数量不一致；
- 功能边界扩大；
- 结构性文案问题。

状态：

```text
review_status = SUPPLIER_REVISION_REQUIRED
open_tasks += TEXT_FIX
```

### 5. AUTO_FIX_PENDING_CONFIRMATION

只存在低风险且允许自动修改的问题。

状态：

```text
review_status = AUTO_FIX_PENDING
```

系统生成建议稿和 Diff，但不得自动进入最终通过。

### 6. PASS_WITH_SUGGESTIONS

合规、准确性和证据均通过，仅有传播或表达优化建议。

状态：

```text
review_status = PASSED
publish_status = READY_WITH_SUGGESTIONS
```

### 7. PASS

无未关闭问题。

状态：

```text
review_status = PASSED
publish_status = READY_TO_PUBLISH
```

---

## 四、问题去重

不同 Agent 可能识别同一问题。

去重键建议：

```text
内容版本 + 字段 + 原文范围 + 规则主题
```

合并时保留：

- 所有命中的规则编号；
- 所有 Agent 来源；
- 最高风险等级；
- 最严格动作；
- 最清晰的原因；
- 不冲突的修改建议。

不得简单删除另一个 Agent 的依据。

---

## 五、冲突处理

示例：

- 品牌 Agent 认为表达自然；
- 准确性 Agent 认为功能未确认。

最终必须按准确性问题转人工，不能因品牌通过而放行。

若同一事实：

- 一个 Agent 判通过；
- 另一个 Agent 判明确错误；

则创建“Agent冲突”人工任务，并保存双方依据。

---

## 六、置信度阈值

建议：

- `>= 0.85`：正常执行规则动作；
- `0.65—0.84`：保留问题，但建议人工抽检；
- `< 0.65`：不得自动阻断，转人工确认。

明确确定性规则命中不依赖模型置信度。

---

## 七、重新审核

供应商提交新版本后：

1. 关闭旧版本未完成任务，但不删除；
2. 创建新内容版本；
3. 重新运行格式校验；
4. 重新运行六个 Agent；
5. 对比旧问题是否解决；
6. 未解决问题继续保留；
7. 新问题创建新记录；
8. 通过后保留完整版本链。

---

## 八、发布条件

必须同时满足：

- 格式校验通过；
- 没有 BLOCK；
- 没有待人工问题；
- 没有待修改文字或素材；
- 所有自动修改均已确认；
- 使用的标准快照有效；
- 最终标题、正文和素材已生成固定版本。
