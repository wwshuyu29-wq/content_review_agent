# 上线前检查清单

## 标准文件

- [ ] 中文文件均为 UTF-8
- [ ] 不存在被生产加载的中英文重复文件
- [ ] 每个 Agent 只有一份全局标准
- [ ] 项目编号在所有 YAML 中一致
- [ ] JSON、YAML 和 JSON Schema 可解析
- [ ] 当前项目标准已发布为固定版本
- [ ] 已生成标准快照和文件哈希

## 规则

- [ ] 确定性规则不会误伤官方 Slogan
- [ ] “最少走路”用户偏好不会被误判为最高级
- [ ] “实测”会触发证据检查
- [ ] 标题测试数量与正文数量可校验
- [ ] 酒店和记忆能力进入人工确认
- [ ] 平台空配置不会产生硬性缺失问题
- [ ] 自动替换不修改数字和测试结果

## Agent

- [ ] 六个 Agent 使用统一输出 Schema
- [ ] Prompt 只加载对应标准
- [ ] 不确定时输出 HUMAN_REVIEW
- [ ] 低置信度不自动阻断
- [ ] 模型输出无法解析时有重试和失败状态
- [ ] 每次调用记录模型和 Prompt 版本

## 工作流

- [ ] 原稿不可被覆盖
- [ ] 每次修改生成新版本
- [ ] 自动修改必须人工确认
- [ ] 重新提交后重新审核
- [ ] 仲裁优先级已写入代码
- [ ] 人工操作有审计日志
- [ ] 标准更新不改变历史审核依据

## 测试

- [ ] 准备至少30—50条真实稿件
- [ ] 覆盖通过、修改、人工和阻断
- [ ] 覆盖标题数量不一致
- [ ] 覆盖缺少实测证据
- [ ] 覆盖未确认功能
- [ ] 覆盖绝对化表达
- [ ] 覆盖品牌名称错误
- [ ] 覆盖高广告感但事实正确
- [ ] 记录误报率和漏报率
- [ ] 先运行影子模式，不直接影响真实发布
