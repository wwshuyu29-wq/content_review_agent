# 标准迁移与修改逻辑

本文件用于把现有仓库中重复、过短、路径混乱的标准，迁移为中文唯一标准来源。

---

## 一、迁移目标

迁移后满足：

1. 标题、正文和文件名均使用中文；
2. 每个审核维度只有一份全局标准；
3. 六个 Agent 各自读取固定文件；
4. 项目事实与全局原则分离；
5. 确定性规则与语义规则分离；
6. 每次审核能够追溯标准版本和文件哈希；
7. 标准加载失败时，不允许继续执行审核。

---

## 二、现有目录常见问题

例如同时存在：

```text
内容准确性.md
content_accuracy.md
品牌一致性.md
brand_consistency.md
```

风险：

- 同一 Agent 同时读取两份标准；
- 两份标准逐渐产生不同内容；
- 修改一份后另一份仍被程序加载；
- 审核结果无法解释使用的是哪一份；
- Prompt 上下文重复，增加成本和冲突。

---

## 三、迁移步骤

### 第一步：备份

在修改前完整备份：

```bash
cp -R data/standards data/standards_backup_YYYYMMDD
cp config.json config_backup_YYYYMMDD.json
```

同时记录当前 Git 提交：

```bash
git status
git rev-parse HEAD
```

### 第二步：建立中文唯一文件

全局目录只保留：

```text
合规与广告表达.md
品牌一致性.md
内容准确性.md
实测可信度.md
内容质量.md
传播有效性.md
舆情与素材授权.md
```

项目目录只保留：

```text
项目配置.yaml
官方功能口径.yaml
实测证据要求.yaml
平台发布要求.yaml
项目背景.md
```

### 第三步：归档旧文件

不要立即永久删除，先移动到：

```text
archive/旧标准_YYYYMMDD/
```

归档文件不得位于 Agent 自动扫描目录内。

### 第四步：修改 Agent 配置

将代码中英文路径改成中文路径。

必须使用显式配置，不建议通过“扫描目录并猜测文件用途”的方式加载。

示例：

```json
{
  "accuracy": {
    "global_standard": "data/standards/global/内容准确性.md"
  }
}
```

### 第五步：启动校验

服务启动时必须检查：

- 所有配置文件存在；
- 文件编码为 UTF-8；
- JSON 可以解析；
- YAML 可以解析；
- 同一个 Agent 只配置一个全局标准；
- 旧中英文重复文件未被加载；
- JSON Schema 存在；
- 项目编号一致；
- 标准版本存在。

任一关键标准加载失败时，审核服务应停止启动，而不是静默使用空标准。

### 第六步：生成标准快照

每次项目发布标准时生成：

- `标准快照编号`
- 项目标准版本
- 每个文件 SHA256
- 发布时间
- 发布人
- 适用项目
- 是否启用

审核任务只引用快照，不直接引用会持续变化的“最新文件”。

### 第七步：历史数据兼容

旧审核记录保留原路径和旧版本，不应强行改写。

新审核任务使用新快照。

若需要重跑历史稿件，应创建新的审核运行记录，不覆盖旧结论。

---

## 四、全局标准与项目标准的边界

### 全局标准写

- 应该如何判断绝对化；
- 功能必须有依据；
- 实测需要证据；
- 标题和正文要一致；
- 传播有效性如何评分。

### 项目标准写

- 小度想想具体有哪些能力；
- 哪些能力待确认；
- 官方名称和产品定位；
- 当前项目 Slogan；
- 平台必带要求；
- 项目审核策略。

### 确定性规则写

- 错误名称替换；
- 固定高风险短语；
- 测试数量一致性；
- 触发实测证据检查；
- 结构化字段缺失。

同一个事实不要在三处重复维护。

---

## 五、标准修改发布流程

管理员修改标准时：

1. 编辑草稿；
2. 系统校验格式；
3. 展示修改 Diff；
4. 运行历史测试集；
5. 展示新增命中、消失命中和状态变化；
6. 管理员确认；
7. 发布新版本；
8. 生成标准快照；
9. 新任务使用新版本；
10. 旧任务继续保留原版本。

不允许直接覆盖生产标准而不留历史。

---

## 六、回滚逻辑

若新版本出现大量误报：

1. 停止新审核任务；
2. 将项目绑定回上一标准快照；
3. 不删除已经产生的新审核结果；
4. 标记这些结果使用了问题版本；
5. 对受影响内容创建重跑任务；
6. 修正规则后发布下一版本。

---

## 七、验收标准

迁移完成后应满足：

- 搜索 `content_accuracy.md` 等旧路径无生产引用；
- 每个 Agent 只加载一份全局标准；
- 任意审核结果都可查看标准快照；
- 修改标准不会改变历史结果；
- 中文文件名在 Linux 服务器、Docker 和 CI 中可正常读取；
- 所有路径统一使用 UTF-8 和 `pathlib`；
- 不使用字符串拼接生成跨平台路径。
