# Excel 批量内容审核设计

## 目标

将供应商主入口由单条表单改为批量文件工作流：供应商上传 Excel 和图片 ZIP，系统解析预检、生成批次、批量审核，并同时提供表格视图、单条详情视图和审核结果 Excel 导出。

单条录入保留为内部临时补录，不再作为供应商默认入口。

## 输入文件

### Excel

支持 `.xlsx`，第一行必须为表头。标准模板字段：

| 列名 | 必填 | 说明 |
|---|---|---|
| 供应商内容编号 | 是 | 批次内唯一，用于图片匹配 |
| 活动主题 | 是 | 项目活动主题 |
| 平台 | 是 | 微博、抖音、小红书、B站等 |
| 标题 | 是 | 原始标题 |
| 正文 | 是 | 原始正文/脚本 |
| 图片文件名 | 否 | ZIP 内图片文件名，多个文件用英文逗号分隔；第一版每条最多一张图 |
| 计划发布时间 | 否 | `YYYY-MM-DD` 或 Excel 日期 |
| 备注 | 否 | 供应商说明 |

项目不在 Excel 中重复填写，由上传页面先选择项目。

### 图片 ZIP

- 可选；如果 Excel 任一行填写了图片文件名，则 ZIP 必填。
- 支持 `.zip`；内部仅允许 JPG、JPEG、PNG、WEBP。
- 图片按 Excel 的“图片文件名”精确匹配，不按上传顺序匹配。
- ZIP 内文件名不得重复，不允许目录穿越、软链接和超过限制的解压内容。
- 单图最大 20 MiB；ZIP 最大 200 MiB；最多 500 行内容。
- Excel 未引用的图片报告为警告，不阻塞导入；引用但不存在的图片属于该行格式错误。

## 导入流程

1. 用户选择项目。
2. 下载系统生成的标准 Excel 模板。
3. 上传 Excel 和可选图片 ZIP。
4. 后端解析到临时预览，不创建正式批次。
5. 前端展示：总行数、有效行、错误行、警告、逐行错误字段。
6. 用户点击“确认导入”。
7. 后端创建批次、内容主记录和供应商 V1，并将图片保存到批次目录。
8. 格式不通过的行同样保留在批次内，状态为 `INCOMPLETE/INVALID`，方便统一导出退回原因；只有 `PASSED` 行进入自动审核。

预览结果使用短期 `import_token` 标识，保存在服务器临时目录，确认或超时后清理。Token 不包含原始文件内容。

## Excel 校验

### 工作簿级

- 文件必须能被 `openpyxl` 读取。
- 仅处理第一个工作表。
- 表头不能为空、不得重复，必需列必须存在。
- 文件最多 500 条内容。

### 行级

- 供应商内容编号、活动主题、平台、标题、正文必填。
- 批次内供应商内容编号唯一。
- 标题、正文长度符合当前服务限制。
- 发布时间格式合法。
- 图片文件名安全且在 ZIP 中存在。
- 空白尾行忽略。

预览每行返回：`row_number`、标准化内容、`errors[]`、`warnings[]`、`valid`。

## 审核台视图

### 表格视图（默认）

一行一条内容，显示：

- 内容编号、平台、标题、正文摘要
- 格式状态、审核状态、发布状态
- 问题数量、最高风险、问题摘要
- 最终标题、最终正文
- 当前开放任务
- 操作：查看详情、运行审核、处理任务

支持按项目、批次、三类状态筛选。

### 单条可视化视图

保留现有卡片/详情能力，展示：

- 原稿、图片、版本历史
- Agent 结构化问题、规则、证据、置信度
- 原文与 AI 建议稿 Diff
- 接受、编辑后接受、拒绝、风险人工决定

表格视图和单条视图读取同一套 API 和数据库数据。

## 导出 Excel

支持按批次导出 `.xlsx`。保留供应商原始字段，并追加：

- 系统内容编号
- 批次编号
- 格式校验
- 审核状态
- 发布状态
- 问题数量
- 最高风险等级
- 问题分类
- 命中规则
- 问题原因
- 原文证据
- 修改建议
- 最终标题
- 最终正文
- 最终版本来源
- 是否需要人工
- 是否已人工确认
- 审核模型
- 规则版本
- 审核完成时间

问题、规则、证据和建议有多项时使用换行分隔，并开启 Excel 自动换行。未审核完成的内容也允许导出，状态如实填写。

导出不修改数据库状态。

## API

- `GET /api/import-template`：下载标准 Excel 模板。
- `POST /api/imports/preview`：上传 `project_id + xlsx + optional zip`，返回预览与 `import_token`。
- `POST /api/imports/{token}/confirm`：确认导入，创建批次和内容。
- `GET /api/batches/{batch_id}/export`：下载审核结果 Excel。
- `GET /api/contents/table`：表格视图所需扁平化内容列表。

原有单条/JSON 批次 API 保留用于内部补录和自动化调用。

## 前端

### 供应商上传页

默认展示批量导入：

- 项目选择
- 模板下载
- Excel 拖拽区
- 图片 ZIP 拖拽区
- 解析预览表格
- 错误筛选与确认导入

“单条临时补录”作为折叠区域。

### 审核台

增加“表格 / 卡片”切换，默认表格。表格支持横向滚动、固定标识列、状态筛选和行详情。

### 导出

批次操作区提供“导出审核结果”按钮，直接下载 `.xlsx`。

## 安全与错误处理

- 文件上传流式写入并设置大小限制。
- ZIP 防目录穿越和压缩炸弹；只解压被 Excel 引用且通过格式校验的文件。
- 解析异常返回可读错误，不创建半成品批次。
- 确认导入使用数据库事务；重复确认同一 token 返回原批次，不重复创建。
- 导出使用只读查询和临时文件，响应完成后清理。

## 测试与验收

- 模板可下载并被 `openpyxl` 正常打开。
- 多行 Excel 可预览并显示精确行号错误。
- Excel + ZIP 按文件名正确匹配。
- 缺图、重复编号、错误日期、缺表头被正确标记。
- 确认导入生成一个批次和多条 V1。
- 批量审核后表格和卡片视图结果一致。
- 导出 Excel 包含原始字段、问题、建议和最终稿。
- 后端 pytest、前端 TypeScript/Vite 构建和真实 HTTP 主链路全部通过。
