# 内容审核标准（Review Rubric）

本文档定义图文/视频内容的审核维度、风险分级与处置规则。`auto_review.py` 的判定 prompt 直接引用本文件；人工审核时也应参照本标准填写结论理由。

## 目录

- 风险类别定义
- 风险等级与处置规则
- 图片判定输出格式
- 视频判定说明
- 人工审核操作规范

## 风险类别定义

| 类别 | 说明 | 典型信号 |
|------|------|----------|
| illegal | 违法违规内容（毒品、赌博、武器交易等） | 明确的违法物品/行为展示或引导 |
| porn_vulgar | 色情低俗 | 裸露、性暗示、低俗擦边 |
| violence_terror | 暴力血腥/恐怖 | 血腥画面、极端暴力、恐怖主义符号 |
| ad_diversion | 广告导流/引流 | 二维码引流、联系方式导流、无关商业推广 |
| privacy | 隐私泄露 | 清晰人脸+身份信息、证件号、手机号、住址 |
| copyright | 版权疑似侵权 | 明显盗用他人作品/水印被抹除痕迹 |
| rumor_fake | 谣言/虚假信息 | 摆拍冒充实况、明显编造的信息配文 |
| brand_compliance | 品牌/合规风险 | 违反品牌调性、竞品敏感信息、涉政敏感 |

## 风险等级与处置规则

| 命中类别 | 风险等级 | 处置规则 |
|----------|----------|----------|
| illegal / porn_vulgar / violence_terror | 高 | **强制转人工**，无论模型置信度多高，都不允许自动通过或自动拒绝 |
| privacy / copyright / rumor_fake / brand_compliance | 中 | 置信度 ≥ 0.85 时可自动拒绝或自动通过；否则转人工 |
| 未命中任何类别 | 低 | 自动通过；建议按 5%-10% 比例抽样转人工复核（具体比例由使用团队按内容体量调整） |
| 模型明确表示"无法判断" | - | **强制转人工**，禁止兜底猜测 |

处置优先级：只要命中任一"高风险"类别，即使同时命中"低风险"信号，也按高风险处理。

## 图片判定输出格式

`auto_review.py` 要求审核时返回如下 JSON 结构（由 agent 分析图片后填写）：

```json
{
  "risk_categories": ["porn_vulgar"],
  "risk_level": "high",
  "confidence": 0.92,
  "reason": "画面中存在裸露内容",
  "verdict": "needs_human"
}
```

`verdict` 取值：`pass`（自动通过）/ `reject`（自动拒绝）/ `needs_human`（转人工）。判定必须严格套用上表规则，不能自行放宽。

## 视频判定说明

- 视频按固定间隔抽取关键帧（默认 5 秒/帧，可通过 `extract_frames.py --interval` 调整），逐帧套用图片判定逻辑
- 只要任意一帧命中高风险类别，整段视频判定为 `needs_human`
- **已知盲区**：关键帧抽样无法覆盖音频内容、字幕文字、帧间快速闪现画面。这是自动审核的结构性限制，不是实现缺陷，必须在视频审核结果中固定附加提示："关键帧抽样审核，无法覆盖音频/字幕，建议人工快速抽查完整视频"
- 视频时长超过 10 分钟时，建议将抽帧间隔放宽到 10-15 秒，避免单次审核帧数过多

## 人工审核操作规范

- 认领（claim）后独占该条记录，避免多人重复劳动；超过 24 小时未提交结论自动释放认领（由使用团队的定时任务负责，本 skill 不内置强制释放逻辑，需要时可在 `human_queue.py` 基础上扩展）
- 命中高风险类别的记录，提交结论时**必须**填写理由（`human_queue.py submit` 会校验 `--reason` 非空）
- 人工结论同样只能是 `approved` / `rejected`，不设"部分通过"，避免状态复杂化
- 高风险类别建议二审：本 skill 当前实现为单人审核+留痕，二审流程作为后续可选增强项，不在当前范围内强行实现
