# 文案审核标准（视频脚本 / 图文文本）

本文件是**全局标准**（流程一），长期适用，供审核模型与人工审核共同参照。
项目补充标准另存 `<standards-dir>/<项目名>.md`，管理员可随时修改。
禁用词/推荐表达/必须人工确认关键词等结构化规则存 `<standards-dir>/rules.json`（流程七可沉淀）。

## 审核维度（流程四）

1. **品牌一致性**：品牌名、功能名、调性是否正确；是否符合项目品牌口径；是否出现与品牌定位不一致的表达。
2. **内容准确性**：功能介绍、活动规则、优惠内容、必带标签等是否与审核标准一致。
3. **内容质量**：错别字、语病、重复表达、信息缺失、卖点不清、表达不自然。
4. **高风险 / 无法判断**：
   - 功能无法确认是否匹配
   - 提及的明星、IP、第三方品牌与审核条件不匹配
   - 涉及近 2 周风险舆情的内容（舆情来源本期由管理员手动维护，后续接入微博抓取）
   - 违反广告法的高风险表达：政治敏感、色情/低俗、暴力/血腥、虚假信息、夸大产品功能、版权侵权、人身攻击

## 风险分级与处置

| 风险等级 | 判定 | 处置 |
|----------|------|------|
| high 高风险 | 命中违反广告法类表达 | 只给建议，转人工审核（不自动改写、不自动通过） |
| unknown 无法判断 | 功能/明星/IP/第三方品牌无法确认匹配、涉舆情 | 只给建议，转人工审核 |
| mid 中风险 | 表达合规类，需替换确认 | 只给建议，转人工审核 |
| low 低风险 | 仅错别字/语病/格式等文案质量问题 | 自动改写，写入最终标题/正文后置「通过」 |
| none 无问题 | 未命中任何问题 | 直接「通过」，回填最终标题/正文 |

> 处置优先级：命中任一高风险，即使同时有低风险信号，也按高风险处理。

## 状态机（内容状态）

- 流转/等待态：`已提交`、`待人工审核`、`图片/视频修改`、`待供应商补充`、`需修改`
- 终态：`通过`、`已驳回`、`已删除`

关键规则：
- `已提交` 的行由**格式校验列**决定去向：不为「通过」→ 内容读取；为「通过」→ 批量审核。
- 人工审核后管理员直接置 `通过`/`需修改`/`已删除`，**不回炉自动审核**（避免死循环）。
- 供应商换图或补格式后置 `已提交` 重新进流程；退回累计超过 3 次（MAX_ROUNDS）自动 `已驳回`。
- 直接 `通过` 的行必须回填 `最终标题`/`最终正文`。

## 本期范围

- 审核对象：文本（标题/正文/脚本）为主，图片可接多模态审核（复用 ERNIE-VL）。
- 视频文件本期不做自动画面审核；视频类内容的**脚本文本**照常审核。
- 近 2 周风险舆情：管理员手动维护风险词（写入 rules.json 的必须人工确认关键词），微博抓取后接。
